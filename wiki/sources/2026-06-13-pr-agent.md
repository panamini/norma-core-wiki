# Source: PR Agent

## Summary
Ingested a concise PR execution control prompt that enforces strict verification, zero-assumption edits, and scoped PR delivery for repo changes.

## Key points
- Demands mandatory source verification and explicit `BLOCKED_STATE_UNKNOWN` when key state is unverified.
- Defines a four-step analysis and execution workflow for PR-safe changes.
- Enforces minimal patch scope, explicit risk/impact tracking, and verifiable test/grep evidence.
- Provides stricter language for direct PR execution discipline and scope rollback.
